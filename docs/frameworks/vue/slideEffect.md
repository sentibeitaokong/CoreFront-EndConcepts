# 副作用（Side Effect）

## 1. 什么是副作用？

在计算机科学中，**副作用**（Side Effect）是指一个函数或表达式在执行时，除了返回值之外，还修改了外部状态（如全局变量、DOM、文件系统、发起网络请求等）或依赖外部不可预测的状态。

在 Vue 3 的响应式系统中，副作用通常指那些**依赖响应式数据，并在数据变化时重新执行**的函数，例如：
- 组件的渲染函数（将状态映射到 DOM）
- `watch` 和 `watchEffect` 的回调
- 计算属性（computed）本身是纯函数，但它内部可以包含副作用（不推荐）


## 2. Vue 3 中的副作用类型

### 2.1 组件渲染副作用

每个 Vue 组件实例都有一个渲染副作用（render effect），它负责执行组件的 `render` 函数，生成虚拟 DOM 并更新真实 DOM。当组件依赖的响应式状态（`ref`、`reactive`、`computed` 等）发生变化时，渲染副作用会重新执行，触发组件更新。

### 2.2 `watchEffect` 副作用

`watchEffect` 接收一个函数，该函数内部可以访问响应式数据。Vue 会自动收集其依赖，并在依赖变化时立即重新运行该函数。

```javascript
import { ref, watchEffect } from 'vue'

const count = ref(0)
watchEffect(() => {
  console.log(`count is ${count.value}`) // 自动追踪 count.value
})
// 立即打印 "count is 0"
count.value++ // 打印 "count is 1"
```

### 2.3 `watch` 副作用

`watch` 显式监听一个或多个响应式数据源，并在数据变化时执行回调。它比 `watchEffect` 更精确控制触发时机和获取新旧值。

```javascript
watch(count, (newVal, oldVal) => {
  console.log(`count changed from ${oldVal} to ${newVal}`)
})
```

### 2.4 `computed` 中的副作用（不推荐）

计算属性应该是一个纯函数，仅根据依赖的响应式状态计算新值。如果在计算属性内部执行副作用（如修改其他状态、发起请求），会导致难以调试的问题。Vue 会警告这种用法。

### 2.5 生命周期钩子中的副作用

在 `onMounted`、`onUpdated`、`onUnmounted` 等钩子中执行的操作（如添加事件监听器、启动定时器、操作 DOM）也是副作用。

## 3. 副作用的影响

- **性能**：频繁执行的副作用可能导致性能下降，需要合理控制触发频率。
- **可预测性**：副作用引入外部依赖，使程序行为难以预测，尤其是异步副作用（如网络请求）。
- **内存泄漏**：未正确清理的副作用（如定时器、事件监听）会导致内存泄漏。
- **竞态条件**：异步副作用（如搜索请求）可能因返回顺序错乱导致界面状态错误。

## 4. 副作用的管理方法

### 4.1 自动停止的副作用

- **组件渲染副作用**：当组件被卸载时，渲染副作用会自动停止。
- **在组件 `setup()` 中同步创建的 `watchEffect` / `watch`**：当组件卸载时，这些副作用也会自动停止。

### 4.2 手动停止副作用

`watchEffect` 和 `watch` 返回一个停止函数，调用它可以手动停止副作用：

```javascript
const stop = watchEffect(() => { /* ... */ })
// 手动停止
stop()
```

对于组件渲染副作用，无法手动停止（生命周期由 Vue 管理）。

### 4.3 清理异步副作用：`onCleanup`

`watchEffect` 的回调函数接收一个 `onCleanup` 参数，用于注册清理逻辑。该清理函数会在下一次副作用执行前或副作用停止时调用，非常适合处理竞态条件。

```javascript
watchEffect(async (onCleanup) => {
  let isActive = true
  onCleanup(() => { isActive = false })

  const data = await fetchData()
  if (isActive) {
    // 只有当前副作用是最新时，才更新状态
    state.value = data
  }
})
```

### 4.4 使用 `flush` 选项控制执行时机

通过 `flush` 选项可以控制副作用的执行时机：

| 选项      | 执行时机                   | 适用场景                         |
| --------- | -------------------------- | -------------------------------- |
| `'pre'`   | 组件更新前，DOM 尚未更新    | 默认值，适合大多数场景           |
| `'post'`  | 组件更新后，DOM 已更新      | 需要访问更新后 DOM 时            |
| `'sync'`  | 依赖变化时立即同步执行       | 极少数对实时性要求极高的场景     |

```javascript
watchEffect(() => { /* 操作 DOM */ }, { flush: 'post' })
```

Vue 3.2+ 提供了便捷别名：`watchPostEffect` 和 `watchSyncEffect`。

### 4.5 避免在 `computed` 中使用副作用

始终确保计算属性是纯函数，不修改外部状态。如果确实需要副作用，改用 `watch` 或 `watchEffect`。

## 5. 最佳实践

1. **优先使用 `watchEffect` 自动收集依赖**，当需要精确控制依赖源或获取新旧值时使用 `watch`。
2. **始终清理副作用**：在组件卸载时，确保移除事件监听器、清除定时器、取消网络请求。
3. **处理异步竞态**：在 `watchEffect` 中使用 `onCleanup` 标记过期的异步操作。
4. **避免在 `computed` 中产生副作用**，这会导致不可预测的行为和性能问题。
5. **合理使用 `flush` 选项**：默认 `'pre'` 可满足大部分需求；访问 DOM 时使用 `'post'`；慎用 `'sync'`。
6. **手动停止非组件内创建的副作用**：例如在全局模块或工具函数中创建的 `watchEffect`，需要在适当时机手动调用停止函数。

## 6. 总结

| 副作用类型     | 自动停止（组件卸载） | 手动停止 | 支持清理（onCleanup） | 典型用途                     |
| -------------- | -------------------- | -------- | --------------------- | ---------------------------- |
| 组件渲染       | ✅                    | ❌       | ❌                    | 更新 DOM                     |
| `watchEffect`  | ✅（同步创建时）      | ✅       | ✅                    | 自动追踪依赖的副作用         |
| `watch`        | ✅（同步创建时）      | ✅       | ❌（需自行处理）      | 精确监听并获取新旧值         |
| 生命周期钩子   | ✅                    | ❌       | ❌                    | 执行一次性或清理操作         |
