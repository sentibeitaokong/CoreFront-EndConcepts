# 敏感泄露与供应链安全

前端安全不仅包括运行时的攻防（XSS/CSRF），更涵盖工程化生命周期中的两大暗礁：**构建产物的敏感信息泄露**与**依赖供应链的生态污染**。

## 1. 前端无秘密

任何被打包进入浏览器运行环境的代码、Source Map、静态资源与接口响应，均处于**完全公开**的零信任环境。

> **架构红线**：前端仅允许持有**公开配置**与**短效/受限凭证**。绝对严禁在前端代码或环境变量中硬编码服务端私钥、数据库账密、云服务 Secret Key 或长期核心 Token。

## 2. 前端泄露面全景管控矩阵

| 风险触点        | 核心暴露风险                            | 架构级治理策略                                                                              |
| --------------- | --------------------------------------- | ------------------------------------------------------------------------------------------- |
| **`.env` 变量** | 误将服务端 Secret 注入前端构建产物。    | 明确区分构建时变量与运行时变量；启用白名单前缀（如 `VITE_`）；CI 阶段拦截密钥特征词。       |
| **JS Bundle**   | 硬编码导致内部拓扑或测试后门暴露。      | 代码 Review 拦截硬编码 Token、内网 IP 及特权测试账号。                                      |
| **Source Map**  | `.map` 文件公开导致源码逻辑与注释裸奔。 | **生产环境禁止发布至公网**；仅在 CI 打包后私有化上传至监控平台（如 Sentry）并在产物中剔除。 |
| **接口响应**    | 冗余字段暴露用户隐私 (PII) 或风控标签。 | 服务端严格执行**最小字段下发**与**敏感脱敏**（如手机号 `***`）；严禁前端隐式过滤全量对象。  |
| **日志与异常**  | 监控平台上报导致授权头或业务数据落盘。  | SDK 上报前强制过滤 `Authorization`、完整 Cookie、证件号及支付对象；日志设置严格的生命周期。 |
| **本地存储**    | `localStorage` 被 XSS 直接窃取。        | 严禁存储高权长期凭证；认证态优先使用 `HttpOnly Cookie` 或内存短效 Token。                   |

## 3. NPM 供应链与依赖治理

现代前端高度依赖开源生态，任何一个 NPM 包的投毒、劫持或间接污染，都将获得与业务代码同等的执行权限。

### 3.1 核心威胁模型

- **Typosquatting (域名/包名误导)**：注册拼写相近的恶意包诱导误装。
- **Dependency Confusion (依赖混淆)**：利用内外网同名包的解析优先级，骗取构建机下载公网恶意最高版本。
- **Install Scripts (安装脚本劫持)**：利用 `postinstall` 钩子在 `npm install` 阶段静默执行恶意 Shell。

### 3.2 治理与防御基线

| 治理维度         | 落地规范与控制手段                                                                                                    |
| ---------------- | --------------------------------------------------------------------------------------------------------------------- |
| **依赖锁定**     | 必须全量提交 lockfile（`pnpm-lock.yaml` 等）；CI/CD 强制使用 **Frozen Lockfile** 模式构建，确保双端解析哈希绝对一致。 |
| **私有包隔离**   | 内部组件必须使用组织命名空间（如 `@company/ui`）；配置严格的 `.npmrc` 注册表路由；CI 阻断内部 Scope 回源至公网。      |
| **脚本执行控制** | CI 环境默认追加 `--ignore-scripts`，仅通过白名单机制放行确有构建原生模块需求的底层依赖。                              |
| **升级与审计**   | 依赖大版本升级必须纳入 Code Review；引入新包前核对下载量、维护周期与依赖树深度；定期执行 `pnpm audit`。               |

## 4. 浏览器存储风险

| 存储位置       | 风险                                 | 建议                                    |
| :------------- | :----------------------------------- | :-------------------------------------- |
| localStorage   | 持久化且 JS 可读，XSS 可直接窃取     | 不放长期高权限 Token                    |
| sessionStorage | 页面会话级 JS 可读                   | 只放低敏短期状态                        |
| IndexedDB      | 容量大，适合业务缓存，也可被 JS 读取 | 敏感数据加密或不落盘                    |
| Cookie         | 可自动携带，可能引发 CSRF            | 登录态使用 `HttpOnly; Secure; SameSite` |
| 内存变量       | 刷新丢失，XSS 当前会话仍可读         | 适合短期 access token                   |

## 5. 第三方 CDN 与外部脚本管控

引入第三方脚本（统计、广告、客服）等于将当前页面的完整 DOM 读写权与网络请求权拱手相让。

- **版本锁死与 SRI**：必须使用固定版本号拉取外链，并强制配置 `integrity` 属性校验哈希，防范 CDN 节点被篡改。
- **CSP 强隔离**：通过 `script-src` 和 `connect-src` 将第三方脚本的加载源与数据外发目标限制在极小范围的白名单内。
- **核心区禁入**：支付、资金转账、鉴权设置等高敏页面，严禁引入任何非必须的第三方业务侧脚本。

## 6. 发布前线安全

每次核心版本发布或构建流水线 (CI/CD) 终态前，必须通过以下自动化与人工门禁：

| 检查域              | 门禁验收标准                                                                   |
| ------------------- | ------------------------------------------------------------------------------ |
| **产物扫描 (SAST)** | 构建包 (`dist/`) 扫描无私钥、未脱敏 AK/SK、JWT Secret 遗留。                   |
| **资产隔离**        | `.map` 文件、`.test.ts` 文件未混入公网部署产物目录。                           |
| **依赖一致性**      | Lockfile 无人为篡改痕迹，新增依赖变更通过安全评估。                            |
| **接口与日志**      | 抓包确认核心接口无过度下发，监控网络请求无明文 PII 数据上报。                  |
| **底层防线**        | CSP 指令覆盖完整，跨域/同站凭证 (`SameSite`) 配置合规，外链资源均含 SRI 属性。 |

## 7. 实战建议

- 前端变量默认按公开信息处理，任何密钥都不要进入前端构建。
- 生产 Source Map 上传到监控平台后从公网产物中删除。
- 依赖升级进入代码 review，不把 lockfile 变化当作无意义噪音。
- 对第三方脚本建立资产清单，明确负责人、用途、域名、接入页面。
- 关键业务页面启用更严格的 CSP，不允许随意新增外部脚本。
- CI 增加敏感信息扫描、依赖审计、产物检查，尽量把风险挡在发布前。
