# 企业级鉴权与安全方案

## 1. 全局分层与核心架构

企业级鉴权是一套围绕“**身份可信、权限可控、凭证可撤销、数据可隔离、操作可审计、风险可收敛**”的防御闭环。前端收敛暴露面，后端把守最终边界。

### 1.1 架构分层职责矩阵

| 层级                   | 核心目标           | 关键技术基建 / 解决问题                                                                                       |
| ---------------------- | ------------------ | ------------------------------------------------------------------------------------------------------------- |
| **身份层 (IDP)**       | 确认“**你是谁**”   | SSO 单点登录、OIDC 开放身份验证、SAML 安全断言标记语言、LDAP 轻量级目录访问协议、MFA 多因素认证、账号生命周期 |
| **会话层 (Session)**   | 维持登录状态       | Session, Access Token, Refresh Token, 统一登出, 设备绑定                                                      |
| **权限层 (AuthZ)**     | 限制“**能做什么**” | RBAC (角色), ABAC (属性), 菜单/按钮/接口/数据权限                                                             |
| **租户层 (Tenant)**    | 隔离“**数据归属**” | `tenantId` 隔离, 组织架构隔离, 跨租户鉴权与审计                                                               |
| **接口层 (API)**       | 保障请求安全       | CSRF 防御, CORS 白名单, 签名防重放, 幂等性, 限流                                                              |
| **前端层 (Client)**    | 收敛暴露面         | 路由守卫, 视图权限组件, CSP, 字段脱敏, Source Map 私有化                                                      |
| **工程层 (DevSecOps)** | 确保代码可信       | SAST 静态应用安全测试, 依赖审计, Secret 凭证扫描, CI/CD 质量门禁, 灰度发布                                    |
| **审计层 (Audit)**     | 事后溯源追责       | 登录/操作日志, 风控告警, Trace ID 链路追踪, 异常会话吊销                                                      |

### 1.2 标准 Web 首选链路 (OIDC + BFF)

**请求流转路径**：`Browser -> Web App -> SSO/IdP -> App Backend (BFF) -> Business APIs`

- **步骤**：未登录重定向 IdP ➔ 完成认证 (密码/MFA/企业微信) ➔ 携带 `code` 回调业务前端 ➔ 前端将 `code` 移交 BFF ➔ BFF 校验 `state/nonce` 并在内网换取 Token ➔ 校验 `id_token` ➔ BFF 侧建立 Session 并下发 `HttpOnly Cookie`。
- **BFF 架构价值**：彻底隐藏 IdP Token 及内部微服务 Token，防止浏览器 JS 泄露；在服务端完成用户信息聚合、权限树拼装、数据脱敏，极大减轻前端架构负担。

## 2. 凭证治理与权限模型

### 2.1 凭证生命周期策略

| 凭证类型          | 生命周期 | 推荐存储介质             | 核心管控策略                                                      |
| ----------------- | -------- | ------------------------ | ----------------------------------------------------------------- |
| **Session ID**    | 中短期   | `HttpOnly Cookie`        | Web 后台首选；需设置空闲过期 (如 30min) 与绝对过期 (如 8h)。      |
| **Access Token**  | 短期     | 内存 / Cookie / Header   | 寿命建议 5-15 分钟；绝不建议长期存在 `localStorage`。             |
| **Refresh Token** | 中长期   | `HttpOnly Cookie` / 后端 | **强制轮换 (Rotation)**，绑定设备特征；重用即视为被盗，吊销全族。 |
| **CSRF Token**    | 会话级   | 页面变量 / 自定义 Header | 保护所有写操作；跨站场景 (`SameSite=None`) 下的强制要求。         |
| **Action Ticket** | 极短期   | 后端一次性消费           | 用于登录回调、高敏操作二次确认；通常 5 分钟内单次有效。           |

### 2.2 复合权限模型 (RBAC + ABAC + 数据权限)

- **RBAC (基于角色)**：`用户 ➔ 角色 ➔ 权限点 ➔ 资源`。控制基础的导航展示 (菜单)、路由进入 (页面) 与按钮展示 (操作)。**前端隐藏仅为体验优化，任何写接口必须在网关/服务端重新进行 RBAC 校验。**
- **ABAC (基于属性)**：适用于高风险复杂业务。结合**用户属性** (职级/MFA状态)、**资源属性** (密级/归属)、**环境属性** (内网 IP/受管设备)、**动作属性** 进行动态拦截 (如：非内网 IP 仅允许只读)。
- **数据与租户隔离**：
- 服务端必须从**绝对安全的登录上下文**中提取 `tenantId` 和 `userId`，绝不信任前端传入的身份参数。
- 数据库查询、更新、删除必须强制追加租户与数据范围 (本人/本部门/全公司) 条件。

## 3. 前端防御与接口边界

### 3.1 前端落地基线

- **路由守卫**：启动前拉取 `/me` 与权限树；无权限跳 403，失效跳登录。
- **权限指令 (`v-permission`)**：基于稳定编码控制 DOM 节点挂载。高风险按钮 (如：批量导出/付款) 触发后必须配合后端的二次确认或短时 Ticket。
- **缓存边界**：仅缓存头像、昵称、UI 偏好等低敏数据。**严禁缓存** Refresh Token、私钥、明文 PII (如完整身份证号)、风控标签。
- **工程安全配置**：全站强制 HTTPS (启用 HSTS)；严配 CSP (限制 `unsafe-inline` 与非法外连)；启用 SRI 校验第三方 CDN；设置 `frame-ancestors` 防点击劫持。

### 3.2 接口层攻防矩阵

| 风险类别                 | 核心防护重点                                                                           |
| ------------------------ | -------------------------------------------------------------------------------------- |
| **CSRF (跨站请求伪造)**  | Cookie 配置 `SameSite=Lax` + 强制校验 CSRF Token + 强校验 `Origin/Referer`。           |
| **XSS 后续越权**         | 接口层面收敛暴露面，敏感操作要求 MFA 二次确认；使用 CSP 阻断恶意数据外传。             |
| **越权操作 (水平/垂直)** | 服务端严格执行 RBAC/ABAC 鉴权、数据范围过滤与租户 ID 强制隔离。                        |
| **跨域误配**             | 精确配置 CORS 白名单域名，**绝不使用** `Access-Control-Allow-Origin: *` 配合凭证。     |
| **数据泄露**             | 接口实行**最小字段返回** (剔除内部风控标签与密码哈希)；大批量查询/导出强制分页与限频。 |

## 4. 风险分级与审计监控

### 4.1 动态风险管控

不同操作匹配不同强度的校验策略，避免“**一刀切**”影响业务效率：

- **低风险** (查看列表/改 UI)：登录态 + 基础 RBAC 权限。
- **中风险** (编辑订单/提交审批)：服务端强鉴权 + 记录操作审计日志。
- **高风险** (删除/付款/导出)：短时操作 Ticket / MFA 二次验证 + 详细审计落库。
- **极高风险** (跨租户/改权限模型)：强 MFA + 双人审批授权 + 实时风控告警 + 回滚预案。

### 4.2 审计与链路追踪

审计日志必须剔除 PII 明文、密码与完整 Token，核心记录字段包括：
`traceId` (全链路追踪)、`userId`、`tenantId`、`sessionId` (哈希值)、`action` (操作类型)、`resourceId` (目标资源)、`ip/deviceId`、`riskLevel` (风险评级)、`result` (成功/拦截/失败及原因)。

## 5. 三大典型业务架构落地指南 (推荐组合)

### 5.1 普通企业内部后台

- **认证与会话**：OIDC SSO 登录，BFF 换取 Token；通过 Session + `HttpOnly Cookie` 维持状态。
- **权限与隔离**：标准的 RBAC 体系控制菜单与接口；服务端执行基础的数据权限与租户隔离。
- **基础防御**：`SameSite` + CSRF Token + Origin 校验保护所有写操作；权限变更与敏感操作记入审计。

### 5.2 金融/高安全级别系统

- **零信任准入**：登录强制 MFA，异常环境 (异地/新设备) 触发动态活体或二次验证。
- **短效凭证**：Access Token 极短时效，Refresh Token 每次轮换并与硬件设备指纹强绑定。
- **风控阻断**：高敏页面严格配置 CSP 并实施字段脱敏 (`***`)；大额付款、批量导出全面接入风控策略大盘、触发双人审批机制与实时告警。

### 5.3 多租户 SaaS 平台

- **全维物理/逻辑隔离**：关系型数据库、Redis 缓存、OSS 文件、MQ、搜索索引强制携带租户隔离维度。
- **零信任上下文**：服务端仅依赖解析后的 Token Context 识别租户，绝不接收前端传递的租户标识参数。
- **复合鉴权矩阵**：融合 RBAC + ABAC + 资源归属进行多重鉴权；跨租户管理员操作必须走独立的提权审批工作流并实施最高级别审计。
