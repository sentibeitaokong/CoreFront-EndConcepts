# 性能监控与持续治理

**核心本质**：性能监控不是把 Lighthouse 跑分贴到报告里，而是建立“**发现问题、定位原因、验证收益、防止回退**”的工程闭环。没有线上真实用户数据，优化很容易变成本地机器上的自我安慰。

**治理准则**：实验室数据负责定位，真实用户数据负责裁决；性能预算必须进入发布流程，不能只靠人工自觉。

## 1. 指标体系分层

性能指标需要按用途分层，否则数据会很多但没有决策价值。

* **用户体验指标**：LCP、INP、CLS、FCP、TTFB，回答用户体感是否变好。
* **工程诊断指标**：Long Task、JS 错误率、接口耗时、资源失败率、Bundle 体积，回答问题为什么发生。
* **业务转化指标**：跳出率、停留时长、转化率，回答性能优化是否产生业务收益。
* **发布关联指标**：版本号、构建号、灰度批次，回答是哪次发布引入退化。

## 2. RUM 真实用户监控

RUM（Real User Monitoring）采集真实用户设备、网络和页面上的数据，是性能治理的最终依据。

```javascript
import { onCLS, onINP, onLCP } from 'web-vitals';

function report(metric) {
  const body = JSON.stringify({
    name: metric.name,
    value: metric.value,
    rating: metric.rating,
    id: metric.id,
    path: location.pathname,
    release: window.__APP_RELEASE__
  });

  navigator.sendBeacon('/api/performance', body);
}

onLCP(report);
onINP(report);
onCLS(report);
```

### 上报维度

* **页面路由**：定位哪个页面慢。
* **设备类型**：区分高端机、低端机、移动端、桌面端。
* **网络类型**：区分 4G、Wi-Fi、弱网。
* **浏览器版本**：识别兼容性和内核差异。
* **地区与 CDN 节点**：定位网络和边缘缓存问题。
* **发布版本**：把性能变化与发版记录挂钩。

## 3. 实验室监控

实验室数据可控、可复现，适合做上线前门禁和问题定位。

* **Lighthouse CI**：稳定跑关键页面，追踪分数和指标趋势。
* **Unlighthouse**：批量扫描站点页面，适合文档站和多路由应用。
* **WebPageTest**：模拟不同地区、网络和设备的加载表现。
* **Chrome DevTools**：通过 Performance 火焰图定位长任务、重排和资源瀑布。
* **Bundle Analyzer**：观察构建产物变化，防止依赖膨胀。

实验室数据不等于真实用户数据，但它能稳定复现问题；RUM 数据真实但噪声大，需要按 P75、版本、页面、设备维度聚合。

## 4. 性能预算与发布门禁

性能预算要写进 CI 和发布流程，避免每次迭代都悄悄变慢。

```json
{
  "budgets": {
    "lcp": 2500,
    "inp": 200,
    "cls": 0.1,
    "initialJsKb": 170,
    "initialCssKb": 50
  }
}
```

* **硬门禁**：核心页面 LCP、CLS、Initial JS 超标时阻断发布。
* **软告警**：非核心页面或轻微退化进入告警和排期。
* **趋势追踪**：单次不超标但持续变差，也必须被发现。
* **版本对比**：每次发布后对比上一稳定版本的 P75 指标。

## 5. 问题定位流程

性能问题要避免“看到哪个指标差就随手改哪里”。

* **先确认范围**：是全站慢、单页面慢，还是某类设备慢。
* **再拆指标阶段**：LCP 慢要拆 TTFB、加载延迟、加载耗时、渲染延迟。
* **然后看瀑布流和火焰图**：判断瓶颈在网络、服务端、资源还是主线程。
* **最后做 A/B 验证**：优化上线后按版本和用户分桶确认收益。

## 6. 告警策略

性能告警必须控制噪声，否则团队很快会忽略它。

* **使用 P75 或 P90**：不要用平均值掩盖长尾用户体验。
* **按核心页面告警**：首页、搜索页、详情页、支付页优先级最高。
* **设置持续窗口**：连续多个采样窗口异常再告警，避免偶发抖动。
* **关联错误和接口**：性能退化常常与 JS 错误、接口失败一起出现。

## 7. 核心度量指标

| 核心指标 | 观测目标与含义 | 优化达标线 |
| --- | --- | --- |
| **LCP P75** | 真实用户首屏主体内容速度。 | < 2.5s |
| **INP P75** | 真实用户交互响应速度。 | < 200ms |
| **CLS P75** | 页面视觉稳定性。 | < 0.1 |
| **Error Rate** | JS 错误是否影响体验。 | 持续压低 |
| **Budget Drift** | 性能预算是否持续退化。 | 发布前阻断 |
