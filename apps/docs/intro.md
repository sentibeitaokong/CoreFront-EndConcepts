# 前端核心知识体系 (Core Front-End Concepts)

这个项目围绕“**前端核心概念**”构建了一套从基础语法、浏览器原理、现代框架到工程化实践的系统化知识库。它既包含 HTML、CSS、JavaScript、TypeScript 等基础能力，也覆盖 Vue / React 框架生态、网络协议、设计模式、数据结构与算法，以及组件库和迷你 Vue 源码实现。

## 1. Web 基础技术基石 (The Trinity)

Web 基础是所有前端能力的起点。本项目将 HTML、CSS、JavaScript 拆分为基础篇与进阶篇，帮助你从“**能写页面**”逐步过渡到“**理解浏览器如何执行页面**”。

### 1.1 HTML：结构、语义与 Web API

- **核心作用：** 定义页面内容结构，是浏览器解析、渲染和可访问性的基础。
- **关键内容：**
  - **[HTML 基础结构](/html/basic/htmlBasicStructure)：** 理解文档声明、`head`、`body`、元信息与资源加载。
  - **[常用标签](/html/basic/commonTags)：** 掌握文本、链接、列表、表格、表单等基础标签的语义。
  - **[DOM 事件](/html/basic/domEvents)：** 理解事件绑定、事件流、冒泡、捕获与事件委托。
  - **HTML5 能力：** 包含语义化标签、增强表单、多媒体、Canvas / SVG、拖拽、地理定位、Web Storage、Web Worker、WebSocket、History API 等现代浏览器能力。

### 1.2 CSS：样式、布局与视觉表现

- **核心作用：** 控制页面视觉呈现、布局结构、响应式适配与交互动画。
- **关键内容：**
  - **基础机制：** 选择器、继承、文档流、盒模型、浮动与定位。
  - **现代布局：**
    - **[Flexbox](/css/advanced/layout/flexibleBox)：** 适合一维排列、对齐和组件内部布局。
    - **[Grid](/css/advanced/layout/grid)：** 适合二维网格和页面级布局。
    - **表格布局与多列布局：** 适合特定排版场景的补充方案。
  - **视觉增强：** 颜色、渐变、圆角、阴影、背景增强、文本效果。
  - **动画与变换：** `transition`、`animation`、`transform` 构成基础动效能力。
  - **响应式设计：** 通过单位体系和媒体查询适配不同设备尺寸。

### 1.3 JavaScript：语言核心与运行机制

- **核心作用：** 为页面提供交互逻辑、数据处理、异步控制和复杂业务能力。
- **关键内容：**
  - **基础语法：** 数据类型、变量声明、解构、函数、作用域、闭包、`this`、浅拷贝与深拷贝、相等性判断。
  - **底层机制：** 执行上下文、调用栈、内存空间、原始值包装类型、原型链与继承。
  - **高级数据类型：** 数组、对象、函数、字符串、数字、`Set` / `Map`、正则表达式、操作符。
  - **异步编程：** Iterator、Generator、Promise、Async / Await、事件循环。
  - **元编程与杂项：** Symbol、Proxy、Reflect、严格模式、装饰器、ArrayBuffer、错误处理、跨域与性能优化。
  - **手写实现：** 数组、对象、函数、Promise、Axios、继承、高阶函数和全局方法等源码级练习。

## 2. 语言进阶与类型系统 (TypeScript & RegExp)

当代码规模上升后，类型约束和模式匹配能力会直接影响项目的可维护性。

### 2.1 TypeScript

- **核心作用：** 在 JavaScript 之上提供静态类型检查，提前暴露潜在错误，提升大型项目协作效率。
- **学习重点：**
  - 类型声明、接口、类型别名。
  - 联合类型、交叉类型、泛型、类型收窄。
  - 工具类型与类型推导。
  - 与 Vue、React、组件库、工程化工具链结合使用。

### 2.2 正则表达式

- **核心作用：** 处理字符串匹配、校验、提取与替换。
- **学习重点：** 字符类、量词、分组、断言、贪婪与非贪婪匹配，以及在表单校验、文本解析中的实践。

## 3. 浏览器、网络与安全 (Browser, Network & Security)

前端应用运行在浏览器中，也依赖网络协议完成资源传输。理解这部分内容，才能真正解释页面为什么慢、请求为什么失败、缓存为什么生效。

### 3.1 网络协议体系

- **协议分层：** OSI 模型、IP、TCP、UDP。
- **Web 通信：** DNS、HTTP、HTTPS、请求头与响应头、CDN。
- **核心问题：**
  - DNS 如何把域名解析为 IP。
  - TCP 为什么需要三次握手和四次挥手。
  - HTTPS 如何通过 TLS 保证传输安全。
  - HTTP 缓存、CDN 和请求头如何共同影响资源加载。

### 3.2 浏览器运行机制

- **进程与线程：** 理解浏览器多进程架构、渲染进程、主线程与工作线程的职责边界。
- **渲染流程：** HTML 解析、CSSOM 构建、布局、绘制、合成与页面更新。
- **缓存机制：** 强缓存、协商缓存、缓存头字段和浏览器缓存策略。

### 3.3 Web 安全

- **XSS：** 防止恶意脚本注入，谨慎处理用户输入和动态 HTML。
- **CSRF：** 防止跨站请求伪造，理解 Token、SameSite Cookie 等防护策略。
- **跨域：** 理解同源策略、CORS 与真实业务请求的边界。

## 4. 现代前端框架 (Frameworks & Libraries)

本项目同时整理了 Vue 和 React 的基础用法、生态能力与底层机制，并进一步通过源码实现帮助理解框架设计。

### 4.1 Vue.js

- **基础体系：** 模板语法、响应式基础、计算属性与侦听器、生命周期、组件通信、组合式函数、自定义指令、插件、模板引用。
- **核心生态：** Vue Router、Pinia、构建工具、SSR、异步组件、KeepAlive、Teleport、Transition、Suspense。
- **源码进阶：**
  - 响应式系统：`reactive`、`ref`、`computed`、`watch`、`watchEffect`。
  - 调度系统：`scheduler`、`nextTick`。
  - 渲染器：VNode、`createRenderer`、文本节点、注释节点、Fragment、元素、组件处理。
  - 编译器：parse、transform、codegen。
  - Diff 算法：双端 Diff、快速 Diff、`patchKeyedChildren`。

### 4.2 React.js

- **基础体系：** JSX、元素渲染、组件与 Props、状态与生命周期、事件处理、条件渲染、列表渲染、表单、样式、组件通信。
- **Hooks：** `useState`、`useEffect`、`useMemo`、`useCallback`、`useRef`、`useReducer`、`useContext`、`useTransition`、`useDeferredValue`、`useId`、`useSyncExternalStore` 等。
- **生态能力：** React Router、React Redux、Portal、HOC、动画与性能优化。
- **核心理解：** 单向数据流、不可变更新、虚拟 DOM、Diff、Fiber、并发渲染和副作用管理。

## 8. 推荐学习路径 (Learning Path)

如果你希望按体系推进，可以按照以下顺序学习：

- **打牢基础：** HTML -> CSS -> JavaScript 基础。
- **理解运行时：** JS 执行机制、异步编程、浏览器渲染、网络协议。
- **进入框架：** Vue 或 React 基础 -> 路由 -> 状态管理 -> 组件通信。
- **深入原理：** 响应式、虚拟 DOM、Diff、调度器、编译器。
- **补齐工程化：** TypeScript、构建工具、包管理、Lint、测试、CI/CD、Monorepo。
- **沉淀能力：** 组件库源码、设计模式、数据结构与算法、性能与安全。

## 9. 常见问题 (FAQ) 与学习建议

### 9.1 这个项目适合初学者还是进阶开发者？

两者都适合。初学者可以从 HTML、CSS、JavaScript 基础开始；有经验的开发者可以重点阅读框架源码、工程化、设计模式、算法和组件库实现。

### 9.2 为什么要同时学习 Vue、React 和源码实现？

Vue 和 React 代表了两套主流视图层设计思路。学习 API 能解决业务开发问题，理解源码能解释响应式、渲染、调度、组件更新和性能优化背后的原因。

### 9.3 前端工程化为什么单独成体系？

现代前端项目已经不只是页面开发。依赖管理、构建性能、代码规范、测试、发布、监控、Monorepo 和组件库复用，决定了项目能否长期稳定演进。

### 9.4 算法和设计模式对前端真的重要吗？

重要。框架 Diff、树形菜单、权限系统、虚拟列表、表单联动、缓存策略、状态流转和组件抽象都依赖算法思维与模式化设计。它们能让代码从“**能跑**”走向“**可维护、可扩展**”。
